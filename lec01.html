Part A - What is Flynn's Taxonomy? (2 marks)
Define the taxonomy and what it classifies.
Answer

Flynn's Taxonomy is a classification system for computer architectures based on two dimensions:

  • Number of instruction streams: Single or Multiple
  • Number of data streams: Single or Multiple

It creates four fundamental categories to compare parallel architectures.

Part B - SISD: Single Instruction, Single Data (2 marks)
Define with characteristics and example.
Answer

    • Sequential execution (Von Neumann).
    • No parallelism.
  • Example: Traditional uniprocessor systems (classic single-core CPUs).

Part C - SIMD: Single Instruction, Multiple Data (2 marks)
Define with characteristics and example.
Answer
  • Definition: Single instruction controls multiple processing elements executing on different data simultaneously.
  • Characteristics:
    • Synchronous execution (lockstep).
    • Efficient for data-parallel tasks (e.g., graphics, arrays).
  • Examples: Vector processors, GPUs, CPU extensions (SSE/AVX).

Answer

  • MPP: A scalable system with many processors, where each has its own private memory and runs its own copy of the OS.

  • Separated from outside networks.
  • Typically Headless (No keyboard, mouse, or monitor) to maximize computing density.

Explain Amdahl's Law and its implications for parallel computing.

S = 1 / (f_s + f_p/N)
1. State Amdahl's Law and explain what it predicts (2 marks)
Answer

Statement: Amdahl's Law places a strict theoretical limit on the speedup that can be realized by using multiple processors.

Prediction: It predicts that the maximum speedup is fundamentally limited by the serial fraction of the code, regardless of how many processors are added.

3. Explain why "it takes only a small fraction of serial content to degrade parallel performance" (3 marks)
Answer

Even a very small serial fraction (e.g., 1%) creates a bottleneck because all processors must wait for the serial part to complete. As N approaches infinity, the term f_p/N becomes zero, and the maximum speedup becomes 1/f_s. So, 1% serial limits speedup to max 100x, even with millions of processors.
